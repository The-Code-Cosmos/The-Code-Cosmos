## The Origin of CodeCosmos: From Blocks to Branches 🚀

### **Level 1: The Scratch & MakeCode Era**
My journey started early on with a big goal: I wanted to build an *Among Us* clone. I began with **Scratch**, but I quickly realized that even "simple" block coding could get complex! 🧩

I eventually moved to **Microsoft MakeCode Arcade**. I pivoted from my *Among Us* idea to a clicking game inspired by *Cookie Clicker*. Using Gemini AI as my co-pilot, I pushed MakeCode to its limits. By my **9th iteration**, I had built a game with:
* **Shops** 🛒
* **Animations** ✨
* **Local saving** 💾

But I wanted more!

---

### **Level 2: Discovering "Real" Code**
One day, I accidentally triggered **Canvas Mode** in Gemini. Instead of blocks, I saw it: **HTML, CSS, and JavaScript**. At first, it looked like gibberish, but then it clicked! 💡

I asked Gemini to build a clicking game using these "real" languages, and suddenly I was looking at **JSX and React**. I realized that while MakeCode was a great sandbox, this new world of web development could connect to **Google Firebase**. I could have:
* Cloud saving ☁️
* User authentication 🔑
* Real databases! 📊

---

### **Level 3: Moving to the Pro Tools**
I eventually hit a wall with the AI's built-in editor—it would sometimes "forget" my code or break my work. I knew I had to go **"Pro."** * **VS Code:** I downloaded a professional code editor.
* **The Terminal:** I learned how to use the command line to run `npm` and `Vite`.
* **The Stack:** I built my codebase using **Vite**, **React**, and **Lucide-React** for icons. 
* **GitHub:** I manage my repositories via a family-supervised account to keep my projects safe and organized.

---

### **Level 4: Launching CodeCosmos.net**
I moved away from free `.web.app` links and decided to create a real brand. I bought **codecosmos.net** via Cloudflare (with help from an adult!) and learned how to configure **DNS settings**, **A records**, and **CNAMEs** to link my Firebase hosting to my own domain. 🌐
